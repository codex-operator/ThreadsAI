# ThreadsAI — документация проекта

`ThreadsAI` — это браузерное расширение для работы с Threads (Meta), которое автоматизирует анализ контента, подготовку ответов и управление пользовательскими настройками через popup-интерфейс и background service worker.

## Что делает расширение

- Считывает контент на страницах Threads через content scripts.
- Парсит ленту/треды и приводит данные к структурам, удобным для обработки.
- Формирует подсказки (prompts) и отправляет запросы через прокси/API.
- Показывает интерфейс управления (popup) и вспомогательные таблицы.
- Хранит настройки и авторизацию в `chrome.storage`.
- Обрабатывает внутренние команды и события через сервис-воркер.

## Общая архитектура

Проект разделён на несколько зон ответственности:

1. **Background layer (`src/background`)**
   - Фоновый сервис-воркер и его расширения.
   - Маршрутизация внутренних сообщений, хранение состояния сессии.
   - Работа с настройками/ограничениями/техническими событиями.

2. **Content layer (`src/content`)**
   - Код, который исполняется на страницах Threads.
   - Извлечение, нормализация и подготовка данных из DOM.
   - Взаимодействие с background по сообщениям.

3. **Popup layer (`src/popup`)**
   - UI расширения (окно по клику на иконку).
   - Авторизация, настройки, управление режимами.
   - Логи и состояние операций.

4. **Table layer (`src/table`)**
   - Отдельный интерфейс отображения табличных данных.

5. **Shared layer (`src/shared`)**
   - Общие константы, утилиты времени, API-конфиг, промпты.

6. **Корневые файлы**
   - `manifest.json` — манифест Chrome Extension (MV3).
   - `src/config.js` — дефолтные настройки и пресеты.

## Структура репозитория и назначение файлов

### Корень

- `manifest.json`
  - Точка входа расширения для браузера.
  - Определяет permissions, background script, content scripts и popup.

- `src/config.js`
  - Базовые настройки расширения (`DEFAULT_SETTINGS`).
  - Дефолтные значения поведения (модель, задержки, режимы и т.д.).

### `src/background`

- `service-worker.js`
  - Главный orchestrator фоновой логики.
  - Обрабатывает входящие сообщения (`chrome.runtime.onMessage`).
  - Держит жизненный цикл операций и координирует взаимодействие вкладки + content script.

- `background-settings.js`
  - Подмешивает в сервис-воркер методы чтения/сохранения настроек.
  - Работает с `chrome.storage` и дефолтными значениями.

- `background-limits.js`
  - Подмешивает метод `decrementFreeParsing`.
  - **В текущей версии лимит отключён**: метод всегда возвращает успешный ответ.

### `src/content`

- `content-bootstrap.js`
  - Стартовый bootstrap content-части.

- `content-script.js`
  - Основная логика интеграции с текущей страницей Threads.

- `content-parser.js`
  - Парсинг контента страницы, извлечение сущностей.

- `content-inline.js`
  - Обработка/генерация inline-взаимодействий.

- `content-settings.js`
  - Слой, связывающий настройки из storage с логикой content.

- `feed-parser.js`
  - Специализированный парсер ленты (feed).

### `src/popup`

- `popup.html`, `popup.css`, `popup.js`
  - Основной интерфейс popup, визуальные стили и поведение.

- `popup-settings.js`
  - Логика управления настройками из popup.

- `popup-prompts.js`
  - Работа с промптами и их выбором/пресетами.

- `popup-parser.js`
  - Интеграция popup-команд с процессом парсинга.

- `popup-logs.js`
  - Отображение и обслуживание логов в интерфейсе.

- `popup-utils.js`
  - Общие утилиты для popup-слоя.

- `auth.js`
  - Функции авторизации, получение/хранение auth-данных.

### `src/table`

- `table.html`, `table.css`, `table.js`
  - Отдельное представление данных в табличном формате.

### `src/shared`

- `api-config.js`
  - Конфигурация API/эндпоинтов и ключей storage.

- `prompts.js`
  - Общие шаблоны и/или константы, связанные с prompt-логикой.

- `time-utils.js`
  - Вспомогательные функции времени и форматирования.

- `constants.js`
  - Глобальные константы проекта.

### Дополнительно

- `src/icons/*`
  - Иконки расширения.

- `rules/cdn_headers.json`
  - Вспомогательные правила/настройки заголовков CDN.

## Как это работает в целом (поток)

1. Пользователь открывает Threads и/или popup расширения.
2. `content-*` скрипты получают контекст страницы и данные из DOM.
3. Popup отправляет команды в background (например, старт операции, запрос настроек).
4. `service-worker.js` маршрутизирует команды и вызывает нужные методы.
5. Для настроек подключается слой `background-settings.js`.
6. Для проверки/декремента лимита используется `decrementFreeParsing` из `background-limits.js`.
7. Результат отправляется обратно в popup/content и отображается пользователю.

## Настройки и хранение данных

Основной storage — `chrome.storage.local`.

Там обычно живут:
- auth-объект пользователя;
- extension key;
- пользовательские настройки;
- кастомные prompt'ы;
- служебные логи/состояния.

## Изменение лимита в этой версии

В этом репозитории лимит бесплатных парсингов отключён на уровне расширения:

- `src/background/background-limits.js` возвращает успешный ответ для `decrementFreeParsing`.
- Поле `parsing_left` отдаётся как `-1` (условно безлимит).
- Причина ответа: `limit_disabled`.

## Локальный запуск (как расширение)

1. Открыть `chrome://extensions`.
2. Включить `Developer mode`.
3. Нажать `Load unpacked`.
4. Выбрать папку проекта (`/workspace/ThreadsAI` или вашу локальную копию).
5. Проверить работу popup и сценариев на Threads.

## Краткое резюме

- `service-worker` — центральная шина событий и команд.
- `content` — чтение и подготовка данных со страницы Threads.
- `popup` — пользовательская панель управления.
- `shared` — общая инфраструктура.
- Лимит на 10 действий в день отключён в `background-limits`.
